# Python dependencies

`requirements.in` contains the direct Python dependencies installed in the VM.
Every dependency must use an exact `==` version.

`requirements.lock` contains the complete transitive dependency tree. It is
specific to CPython 3.12 on Linux amd64 and permits exactly one wheel and one
SHA-256 hash for every package. Do not edit this file manually.

## Adding or updating a package

1. Edit `requirements.in`. Add a direct dependency or change its exact version:

   ```text
   requests==2.34.2
   new-package==1.2.3
   ```

2. From the repository root, regenerate the lock:

   ```bash
   src/rootfs/files/configs/python/update-lock.sh
   ```

   The command requires Docker and network access to PyPI. It resolves the
   complete dependency tree inside a pinned Python 3.12/linux-amd64 container,
   downloads the selected wheels, calculates their SHA-256 checksums, and
   replaces `requirements.lock`.

3. Review both files:

   ```bash
   git diff -- \
     src/rootfs/files/configs/python/requirements.in \
     src/rootfs/files/configs/python/requirements.lock
   ```

   Check that expected direct and transitive versions changed. A package
   removal may also remove dependencies that are no longer used.

4. Run a VM build. `install_python_deps.sh` installs with
   `--require-hashes --only-binary=:all:`. The build fails if a package version,
   wheel, or checksum differs from the lock.

Do not add unpinned requirements, version ranges, source archives, or extra
`pip install` commands elsewhere in the build scripts. Update `requirements.in`
and regenerate the lock instead.
